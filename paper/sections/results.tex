\section{Results}
\label{sec:results}

\subsection{Experimental Setup}
\label{sec:setup}

We evaluate three models across two autonomy levels, each with a
5-hour time budget and 5 random seeds (except Claude Code, which uses
single runs from prior work):

\begin{itemize}[nosep]
    \item \textbf{Claude Code} (Anthropic, frontier): Rich CLI agent
    with Read, Write, Grep, Glob, and Bash tools. Runs via
    \texttt{claude -p} with tool allowlisting.
    \item \textbf{Qwen~2.5 Coder~32B} (Alibaba, AWQ INT4): Served via
    vLLM on AMD MI250X. Structured-output prompting with
    \texttt{ACTION:/ARGS:} parsing.
    \item \textbf{Llama~3.3~70B} (Meta, AWQ INT4): Same vLLM
    infrastructure as Qwen.
\end{itemize}

The two autonomy levels are:
\begin{itemize}[nosep]
    \item \textbf{Full optimizer}: The LLM writes an arbitrary
    \texttt{optimize()} function. It can use the provided
    \texttt{topfarm\_sgd\_solve} solver, implement custom gradients,
    or any combination.
    \item \textbf{Schedule-only}: The LLM writes only
    \texttt{schedule\_fn(step, total\_steps, lr0, alpha0)} returning
    four scalars $(lr, \alpha, \beta_1, \beta_2)$ per step. A fixed
    skeleton handles initialization, gradients, and Adam updates.
\end{itemize}

All three models beat the 500-multi-start baseline on training AEP.
Claude Code achieves the highest held-out improvement (+21.8\,GWh),
while Qwen and Llama achieve modest but positive generalization
(+4.7 and +5.3\,GWh respectively). The winning strategies across all
models share a common structure: they wrap \texttt{topfarm\_sgd\_solve}
with modified initialization (hexagonal lattice, wind-aware clustering)
and tuned hyperparameters (higher learning rate, standard Adam momentum).

\paragraph{Train--test divergence.}
For Qwen, the script with the highest training AEP (5549.5\,GWh) has
a ROWP score of only 4244.0---\emph{below} the baseline. The script
with the best ROWP (4251.4) has a lower training score (5547.9). This
divergence confirms that training AEP alone is an unreliable proxy for
generalization, motivating the held-out evaluation design.

\subsection{Schedule-Only: The Scaffold Confound}
\label{sec:results_schedule}

The schedule-only results reveal a critical confound between model
capability and agent scaffolding.

\paragraph{Initial results (no codebase context).}
When Qwen and Llama run schedule-only mode with the VLLMRunner's
minimal tool interface, both models produce schedules stuck far below
baseline:

\begin{table}[h]
\centering
\small
\caption{Schedule-only mode \emph{without} codebase context injected.
Models wrote schedules without reading the skeleton or baseline
schedule. All ROWP attempts for Qwen were infeasible.}
\label{tab:sched_no_context}
\begin{tabular}{lccc}
\toprule
\textbf{Model} & \textbf{Attempts} & \textbf{Best Train} & \textbf{ROWP Feasible} \\
\midrule
Claude Code         & 320 & 5600.0 & 100\% \\
Qwen 32B (seed 5)  & 54  & 5540.3 & 37\% (best feas: 4121) \\
Llama 70B (seed 2)  & 9   & 5532.6 & TBD \\
\bottomrule
\end{tabular}
\end{table}

Investigating the agent logs reveals the root cause: \textbf{the models
never read the relevant source code.} Llama's turn~1 response says
``Read the skeleton and seed schedule'' but immediately submits code
instead of calling \texttt{read\_file}. Qwen reads the skeleton once
but never reads the baseline schedule. Neither model discovers the
critical alpha--LR coupling ($\alpha_t \propto 1/\eta_t$) that ensures
constraint feasibility at convergence.

By contrast, Claude Code's rich CLI agent reads \texttt{skeleton.py},
\texttt{seed\_schedule.py}, and \texttt{pixwake/optim/sgd.py} before
writing its first schedule, giving it access to the penalty ramping
mechanism.

\paragraph{Enriched prompt (codebase context injected).}
To isolate the effect of information access from model capability, we
inject the contents of \texttt{skeleton.py} and
\texttt{seed\_schedule.py} directly into the VLLMRunner system prompt,
along with an explicit note about the alpha--LR coupling. Re-running
with the enriched prompt:

\begin{table}[h]
\centering
\small
\caption{Schedule-only mode \emph{with} codebase context injected into
the system prompt. Qwen's performance jumps dramatically, matching its
full-optimizer results on training AEP. ROWP backfill pending for
enriched runs.}
\label{tab:sched_enriched}
\begin{tabular}{lcccc}
\toprule
\textbf{Model} & \textbf{Context} & \textbf{Best Train} & \textbf{$\Delta$ from no-context} \\
\midrule
Qwen 32B & None     & 5528.0 & --- \\
Qwen 32B & Enriched & \textbf{5547.8} & \textbf{+19.8} \\
\midrule
Llama 70B & None     & 5526.2 & --- \\
Llama 70B & Enriched & 5540.3 & +14.1 \\
\bottomrule
\end{tabular}
\end{table}

The enriched prompt produces a +19.8\,GWh swing for Qwen---closing the
gap to its full-optimizer performance entirely. This demonstrates that
the schedule-only failure was \emph{information asymmetry}, not model
incapability: when given the same codebase context that Claude Code
reads via its tools, Qwen~32B produces competitive schedules.
