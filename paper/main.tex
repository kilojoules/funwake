\begin{abstract}
We present FunWake, an evaluation framework for LLM agents that write
wind farm layout optimizers. An LLM receives a wake simulation API and
a training farm; a held-out farm with different turbine, polygon, and
wind resource is scored silently after the agent finishes. We compare
three models (Claude Code, Qwen~2.5 Coder~32B, Llama~3.3~70B) across
two autonomy levels---full optimizer and schedule-only---and find that
performance depends critically on \emph{agent scaffolding}, not just
model capability. Smaller models that never read the optimization
codebase (due to limited tool use) produce schedules stuck 13\,GWh
below baseline. When the same codebase context is injected into the
prompt, Qwen~32B immediately matches its full-optimizer performance
(+7\,GWh over baseline), demonstrating that the gap is information
access, not reasoning ability. On the held-out farm, Claude Code's
schedule-only mode generalizes best (+24.8\,GWh), while open-source
models generalize better in full-optimizer mode (+5\,GWh). The optimal
autonomy level is model-dependent, and evaluation frameworks that do
not control for scaffolding quality will misattribute scaffold effects
to model capability.
\end{abstract}

\section{Conclusion}
\label{sec:conclusion}

We introduced FunWake, an agentic coding benchmark for wind farm layout
optimization with held-out evaluation on a physically distinct farm.
Three findings emerge:
(1)~Agent scaffolding---whether the model reads the codebase before
writing code---explains more performance variance than model scale in
the constrained schedule-only setting.
(2)~The optimal autonomy level is model-dependent: frontier models
benefit from constraint (schedule-only), while smaller models need the
safety net of an existing solver (full optimizer).
(3)~Held-out evaluation with different physical parameters is essential;
training AEP gains of +9\,GWh can correspond to held-out losses.

FunWake is open-source and designed to be extended with additional
models, autonomy levels, and held-out farms.
